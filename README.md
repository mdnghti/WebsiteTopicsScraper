# Smart Business & Contact Scraper

Асинхронный парсер для сбора контактных данных (email) и автоматической классификации тематики (ниши) веб-сайтов с использованием локальных LLM (через Ollama). 

Скрипт читает Excel-файл со списком доменов, обходит их главные страницы (а также страницы контактов, «О нас», страницы агентств и XML Sitemap), извлекает email-адреса, а затем определяет категорию бизнеса с помощью нейросети по содержимому сайта (текст, заголовки, метаданные).

## Основные возможности

- **Высокая производительность (asyncio)**: Асинхронная обработка сайтов с пулированием контекстов браузера (Playwright) и подключений. Парсинг разбит на пакеты (батчи) с очисткой памяти (`soup.decompose()`, `gc.collect()`) для исключения утечек.
- **Трехуровневый обход защит**:
  1. **aiohttp** — приоритетный, быстрый метод. Ротация **User-Agent** для каждого запроса.
  2. **cloudscraper** — фоллбек для обхода базовых проверок HTTP с автоматической ротацией User-Agent.
  3. **Playwright** — использование `Chromium` с `stealth`-скриптами для обхода сложной защиты (Cloudflare, Beacon).
- **Интеллектуальный парсинг и валидация**:
  - **MX-валидация**: Все найденные email проходят проверку на наличие MX-записей домена в параллельном режиме (`asyncio.gather`).
  - **Контекстный поиск (Role-based)**: Сбор 200 символов текста вокруг email для определения роли (SEO, реклама, редакция).
  - **Robots.txt**: Скрипт проверяет `/robots.txt` перед началом сканирования для соблюдения правил вежливости.
- **Ранжирование email**: Многоуровневая система приоритетов (8 уровней), учитывающая доменное соответствие, ключевые слова в адресе (sales, advertising) и наличие ключевых слов в окружающем тексте (role context).
- **Анализ ниши (LLM)**: Интеграция с OpenAI API для работы с локальной LLM (llama3.2:3b через Ollama). Нейросеть классифицирует бизнес по Title, Description, заголовкам и контенту.

## Требования

- Python 3.9+
- Установленные библиотеки (см. `requirements.txt`: `pandas`, `beautifulsoup4`, `aiohttp`, `cloudscraper`, `playwright`, `openai`, `dnspython`).
- Запущенный сервер [Ollama](https://ollama.com/) с моделью `llama3.2:3b`.

## Установка

1. Склонируйте репозиторий:
   ```bash
   git clone <URL_репозитория>
   cd websites_topics
   ```

2. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

3. Установите браузеры Playwright:
   ```bash
   playwright install chromium
   ```

4. Запустите Ollama и скачайте нужную модель (по умолчанию используется `llama3.2:3b`, но можно заменить в `main.py`):
   ```bash
   ollama run llama3.2:3b
   ```

## Использование

1. Подготовьте **`sites_to_check.xlsx`**. Поместите список URL в первую колонку или колонку `URL`. Скрипт автоматически выполнит **дедупликацию по каноническому домену**.

2. (Опционально) Настройте переменные окружения для работы:
   - `OPENAI_BASE_URL`: Базовый URL для LLM (по умолчанию `http://localhost:11434/v1` для Ollama).
   - `SCRAPER_CONCURRENCY`: Количество конкурентных потоков сканирования (значение по умолчанию `2`).

3. Запустите парсер:
   ```bash
   python main.py
   ```
4. Скрипт поддерживает работу в виде скомпилированного **.exe** (Nuitka/PyInstaller). Благодаря динамическому определению `BASE_DIR`, все файлы (`sites_to_check.xlsx`, `leads_result.xlsx`) будут искаться и создаваться в той же папке, где лежит исполняемый файл.

### Формат выходного файла (`leads_result.xlsx`)

- **Site URL** — Нормализованный адрес проверенного сайта.
- **Email** — Лучший из найденных email-адресов (согласно 8-уровневой системе ранжирования).
- **Thematic** — Определенная нейросетью тематика (например: News, Law Firm, E-commerce) или статус ошибки (`Offline / Unreachable`, `Error / BatchCrash`).

## Под капотом (Оптимизации)

1. **Параллельная MX-валидация**: Все найденные email-адреса проверяются на наличие MX-записей домена в параллельном режиме через `asyncio.gather`. Это исключает попадание в результат битых или несуществующих ящиков без ущерба производительности.
2. **Контекстный анализ (Context-Aware Extraction)**: Скрипт извлекает текст вокруг email (200 символов до и после) и ищет в нем ключевые слова ролей (CEO, Advertise, Editor). Это радикально повышает точность выбора "лучшего" контакта.
3. **Agency Email Detection**: Глубокий поиск ссылок на рекламные агентства и медиа-киты с выделенным таймаутом (20с), чтобы не блокировать основной поток.
4. **Управление памятью**: Использование `.decompose()` для BeautifulSoup и периодический запуск `gc.collect()` гарантируют, что скрипт может обрабатывать десятки тысяч доменов в одну сессию без переполнения RAM.
5. **Бесконтрольный обход CSS/Image**: Playwright программно блокирует загрузку стилей, медиа, картинок и шрифтов для экономии трафика и ускорения рендеринга.
